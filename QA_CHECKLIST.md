# QAチェックリスト

オリジナル3DカートレースWebアプリの最終確認用チェックリスト。短時間で確認できるよう、優先度ごとに確認する。

## P0: 必須

- [ ] P0-01 起動方法が明記され、クリーン環境で `npm install` から起動まで再現できる。
- [ ] P0-02 開発サーバー起動後、ブラウザで初回ロードしても致命的なコンソールエラーが出ない。
- [ ] P0-03 タイトルまたはメニューからレースを開始でき、プレイ画面へ遷移する。
- [ ] P0-04 3Dのコース、カート、ライバル、カメラ、ライトが表示され、画面が空白にならない。
- [ ] P0-05 加速、ブレーキまたはバック、左右ステアリングがキーボードで操作できる。
- [ ] P0-06 コース外や壁への接触後も復帰でき、進行不能にならない。
- [ ] P0-07 周回、チェックポイント、ゴール、順位または完走判定が正常に動く。
- [ ] P0-08 ドリフト操作ができ、条件を満たすとブーストが発動する。
- [ ] P0-09 CPUカートが複数台出走し、コースに沿って走行する。
- [ ] P0-10 アイテムを取得、保持、使用でき、使用時にゲームが止まらない。
- [ ] P0-11 HUDに最低限の周回、順位、タイム、所持アイテム、速度またはブースト状態が表示される。
- [ ] P0-12 PCブラウザのChromeまたはEdgeで1レースを最後まで完走できる。
- [ ] P0-13 モバイル幅でも画面が破綻せず、タッチ操作で走行とアイテム使用ができる。
- [ ] P0-14 既存IPの名称、キャラクター、コース、音楽、UI演出を直接コピーしていない。

## P1: 重要

- [ ] P1-01 3D見た目がオリジナルの世界観としてまとまり、車体、路面、背景の判別がしやすい。
- [ ] P1-02 カメラが走行中の進行方向とコースを追いやすく、急なめり込みや反転が起きにくい。
- [ ] P1-03 ドリフト中の火花、ゲージ、色変化などでブースト準備状態が分かる。
- [ ] P1-04 ブースト発動時に速度、エフェクト、音のいずれかで爽快感が伝わる。
- [ ] P1-05 CPUが大きく詰まり続けず、最低1周以上は自然にレースへ参加する。
- [ ] P1-06 アイテムに複数種類があり、加速、妨害、防御など効果の違いが分かる。
- [ ] P1-07 アイテム命中、被弾、無敵、復帰などの状態変化がHUDや演出で理解できる。
- [ ] P1-08 HUDがプレイを邪魔せず、デスクトップとモバイルの両方で文字や数値が読める。
- [ ] P1-09 BGM、走行音、ドリフト、ブースト、アイテムなど主要な音が鳴る。
- [ ] P1-10 音量調整またはミュート手段がある。
- [ ] P1-11 リスタート、ポーズ、レース終了後の再挑戦ができる。
- [ ] P1-12 ChromeとFirefoxの最新安定版で基本プレイを確認している。

## P2: 仕上げ

- [ ] P2-01 モバイルの縦向きと横向きで、操作ボタンが重ならず押しやすい。
- [ ] P2-02 ゲームパッド対応がある場合、接続、操作表示、未接続時の挙動が自然である。
- [ ] P2-03 ローディング、アセット読み込み失敗、WebGL非対応時に分かりやすい表示がある。
- [ ] P2-04 1レース中に著しいフレーム落ち、メモリ増加、入力遅延が続かない。
- [ ] P2-05 影、粒子、看板、地形などの演出がプレイ視認性を損なっていない。
- [ ] P2-06 ベストタイム、順位結果、ラップタイムなどの記録表示がある。
- [ ] P2-07 操作説明が短く確認でき、初回プレイヤーが迷わず走り出せる。
- [ ] P2-08 SafariまたはiOS Safariでロード可否と最低限の表示を確認している。
